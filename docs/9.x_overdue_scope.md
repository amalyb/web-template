   `transition/privileged-apply-late-fees` and

## 9.0 Late-fee charging audit (P0 — blocker)

### Finding
`applyCharges()` in `server/lib/lateFees.js` calls:

- `transition/privileged-apply-late-fees` (SCENARIO A: returned late)
- `transition/privileged-apply-late-fees-non-return` (SCENARIO B: never returned)

Neither transition is in the current `ext/transaction-processes/default-booking/process.edn`.
Only the `_backup_*` copies contain them. Grepping the live file returns only
`complete-replacement`.

### Impact
- No `late-fee` line items land on the transaction.
- `lastLateFeeDayCharged` never advances (the transition errors before
  protectedData is updated), so overdue reminders keep firing but
  `chargeHistory` stays empty.
- Borrowers receive the "$15/day fee is being charged" SMS, but are not
  actually billed. Legal / trust exposure.

### Fix plan
1. Restore both transitions to the live `process.edn` from the backup:
   - `transition/privileged-apply-late-fees` (actor: customer, privileged,
     `action/update-protected-data` + `action/privileged-set-line-items` +
     `action/stripe-capture-payment-intent` or the equivalent already in
     the backup).
   - `transition/privileged-apply-late-fees-non-return` (same shape, from the
     `accepted` state).
2. Push with `flex-cli process push --process=default-booking
   --path=ext/transaction-processes/default-booking`.
3. Flip the release alias in the Sharetribe Console to the new version.
4. Version-gate `applyCharges()` the same way `sendAutoCancelUnshipped.js`
   does: read `tx.attributes.processVersion` and skip if it's older than the
   version with these transitions.
5. Dry-run in staging with a real booking + a $0.50 fee override before
   rolling to prod.
6. Add a Slack/log alert when `applyCharges()` throws with error code
   `unknown-transition` so this class of silent failure is caught next time.

### Acceptance
- A test tx with a 2-day-late scan gets 2 `late-fee` line items billed.
- `chargeHistory` accumulates correctly.
- Day-6 hard-stop still holds (max 6 chargeable days of fees at $15/day =
  $90 max via automated path, replacement remains manual).

## 9.1–9.6 Message copy rewrite

Keep 6 messages. Soften Day 1. Normalize preamble ("Sherbrt 🍧"). Ensure
every message uses the short label link via `shortLink()`.

### Day 1 (rewrite — replaces current harsh copy)

Current: `📦 Your Sherbrt return is now late. A $15/day late fee is being charged until your item is scanned in. If the item isn't shipped within 5 days, you may be charged the full replacement value. Please ship it back as soon as possible using your QR code or shipping label: ${shortUrl} 💌`

New:

```
⚠️ Sherbrt 🍧: Your return for "${itemTitle}" ended yesterday. $15/day late fees may now apply until scanned by the carrier. Please ship today using your QR/label: ${shortUrl}. Details: ${termsLink}.
```

Notes:
- Pull `itemTitle` from the included listing (same pattern as step 7/8).
- `termsLink` should be a short link to the Sherbrt return-policy / terms
  page (not a raw URL). Store the canonical URL in a const and short-link
  it once per run.

### Day 2–4, 5, 6

Keep current structure. Apply same formatting changes:
- Prefix each with `Sherbrt 🍧:` once (Day 2–4 currently don't).
- Pass all URLs through `shortLink()`.
- Interpolate `${itemTitle}` where the message can benefit (Day 5, 6).

### Timing / cadence
- Cron hourly. Per-tx per-day idempotency guard unchanged.
- Hard stop after Day 6 unchanged.
- `AUTO_REPLACEMENT_ENABLED = false` unchanged — replacement is
  operator-triggered.
